# Exercicios_POO_Csharp-
Atividades feitas na disciplina de Programação Orientada a Objetos feito em C#.

# Ex1
01) Crie um programa simples que leia uma quantidade N de valores inteiros definida pelo usuário. Após a leitura, o programa deve mostrar: 
	• A soma e a média destes números; 
	• Quantos são pares e quantos são ímpares; 
	• Quantos são maiores que a média; 
	• Quantos são menores que a metade da média;


02) Um Livro armazena seu título, autor e número de páginas. Um livro com até 90 páginas é considerado curto; de 91 a 200 páginas, médio; acima de 200 páginas, longo. Cada livro pode receber avaliações entre 0 e 5 estrelas pelas pessoas que já o leram e, assim, calcular sua avaliação média. Crie um modelo UML para este problema. Escreva os códigos dos métodos que classificam o tamanho do livro e registre uma avaliação para ele.
	

03) Precisamos de uma classe Hora para ser utilizada em sistemas diversos. Uma hora deve ser representada com horas, minutos e segundos. Só podem ser criadas horas válidas. Um objeto Hora pode ser incrementado em minutos, sem modificar o valor original e retornando uma nova hora válida. Um objeto Hora pode ser comparado com outro para verificar se seu horário está mais adiante e, quando solicitado pelo usuário, deve ser exibido no formato HH:MM:SS Crie um modelo UML para a classe proposta. Escreva os códigos dos métodos para incrementar uma hora com minutos e para verificação do horário adiante.
		

04) Uma conta bancária é identificada pelo seu número de 5 dígitos e pelo CPF de seu proprietário. As contas bancárias permitem operações de saque e depósito. A operação de saque tem uma regra especial, permitindo que o cliente faça um saque acima do seu saldo – o que é chamado de limite. Por exemplo, se uma conta tem saldo de R$200 e limite de R$100, seu cliente pode sacar até R$300. O saldo fica negativo e, no próximo depósito, a conta cobrará uma taxa de 3% sobre este valor negativo, descontando do próprio valor de depósito. Para a situação apresentada: a) modele uma classe utilizando UML; b) implemente o código do método ou métodos que realizam a ação de depósito.
	

05) Em uma escola de cursos livres, alunos podem se matricular em um curso por vez, sem verificação de pré-requisitos ou outras condições. O aluno é identificado por seu nome e número de matrícula. Independente do curso, cada aluno faz 4 avaliações por semestre. Cada avaliação vale 25 pontos e, portanto, a soma das avaliações é de 100 pontos. Toda vez que o aluno falta a uma aula, esta falta deve ser registrada. O aluno será aprovado se a nota final for maior ou igual a 60 e tiver frequência de no mínimo 75% das aulas. O modelo e a implementação desta classe já foram feitos por uma equipe de desenvolvimento que você conhece, de acordo com a UML abaixo: Agora, a escola quer conseguir agrupar os alunos nas turmas dos cursos e obter algumas respostas sobre o desempenho como um todo. Independente do curso, cada turma tem uma aula por semana e pode ter até 25 alunos. A turma precisa guardar o nome do curso à qual ela se refere e seu código de horário. Este código é formado de acordo com o dia da semana e o turno. As aulas são de segunda à sexta; segunda feira tem código 2, terça código 3 e assim por diante. Os turnos informam a outra parte do código: 1 para manhã, 2 para tarde, 3 para noite. Uma turma de quinta feira de tarde, por exemplo, tem código de horário 52. Outras informações foram levantadas: • Podem existir turmas com duração entre 8 e 18 semanas, inclusive; • Deve ser possível localizar um aluno de uma turma, usando seu número de matrícula; • Deve ser possível, utilizando a turma, lançar uma nota ou uma falta para um aluno; • A turma deve calcular a nota média de seus alunos; • A turma deve mostrar sua porcentagem de alunos aprovados; • A turma deve emitir um relatório com seu nome, horário e informação detalhada de cada aluno. Para a situação apresentada: a) modele a classe Turma utilizando UML. Não se preocupe com o relacionamento da Turma com Aluno neste momento. Não é necessário incluir métodos construtores nem métodos de acesso na resposta; b) implemente o código do que lança uma nota para um aluno; c) implemente o código do que calcula a porcentagem de aprovação da turma.
